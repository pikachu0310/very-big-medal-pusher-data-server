# サーバー追加方法
1. `openapi.yaml`に新しいパラメータを追加する。
2. `internal/migration/*_*.sql`に新しいパラメータをDBに追加するSQLを追加する。
3. `tools/tools.go`を実行してコードを生成する。
4. `internal/domain/data.go`に新しいパラメータを追加する。
5. `internal/repository/data.go`の`InsertGameData`に新しいパラメータを追加する。

## 注意事項など
既存のデータベース構造を DATABASE.md を読むことで確認し、理解したうえで既存のmigartionファイルを全部見てどうデータを追加すればよいのかを理解したうえで、migrationファイルを書いてください。
動作確認は、docker compose up をしてから実際にデータベースにアクセスしたり、アプリが正常に動いてるかどうかをlogを見ることで確認できます。無限に待機するlogなどのコマンドを実行する際はtimeoutを付けること。編集したら実際にコンテナを起動し、うまく起動できているかどうかの確認までし、できたらコミットプッシュをしてください。動作確認が終わったらコンテナはdownしてください。
プレースホルダー数の追加し忘れがたまにあるので注意してください。
改めて要件をまとめると、DATABASE.mdを見て現状のDB定義を理解し、以下のデータを追加するmigrationファイルを書いて、DATABASE.mdを手動で更新して新しいDB定義を保つようにしてください。動作確認をしたらコミットとプッシュもしてください。
